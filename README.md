# 변증법 앱: 문제정의부터 문제해결까지

## 1. 문제 정의

### 사용자가 겪는 문제
- 감정이 갑자기 커질 때 바로 정리할 도구가 없다.
- 상담이나 치료를 바로 이용하기 어려운 순간이 많다.
- 긴 설명을 하기 힘든 상태에서는 기존 기록형 앱이 오히려 부담이 된다.
- 감정 조절, 관계 대화, 불안 진정 같은 요구가 섞여 있는데 무엇부터 해야 할지 판단하기 어렵다.

### 핵심 문제
- 사용자는 "지금 이 순간" 바로 쓸 수 있는 짧고 실행 가능한 자기조절 가이드를 원한다.
- 입력 부담은 낮아야 하고, 응답은 과잉 해석 없이 빠르게 현재 상태를 정리해줘야 한다.

## 2. 해결 목표

### 제품 목표
- 한 줄 입력만으로도 현재 감정 상태를 정리할 수 있게 한다.
- DBT 기반 기술을 상황에 맞게 짧게 제안한다.
- 로컬 응답과 AI 응답을 모두 지원해 접근성과 유연성을 확보한다.
- 모바일에서도 바로 설치해 쓸 수 있는 가벼운 웹앱으로 제공한다.

### 성공 기준
- 사용자가 입력 후 바로 다음 행동 1~4단계를 받을 수 있다.
- 위기 상황이 아닌 경우 과도하게 위험 중심으로 반응하지 않는다.
- 위기 신호가 보이면 즉시 안전 안내를 우선한다.
- 별도 앱스토어 배포 없이 웹 배포만으로 사용 가능하다.

## 3. 사용자 시나리오

### 시나리오 1: 감정 과열
- 사용자가 "답장이 안 와서 미치겠어"라고 입력한다.
- 앱은 불안/과열 상태로 해석하고 `TIP` 또는 `STOP` 같은 즉시 진정 기술을 제안한다.

### 시나리오 2: 관계 정리
- 사용자가 "뭐라고 말해야 할지 모르겠어"라고 입력한다.
- 앱은 관계 긴장 상황으로 보고 `DEAR MAN` 기반 문장 정리를 돕는다.

### 시나리오 3: 상태 회복
- 사용자가 "이제 좀 괜찮아졌어"라고 입력한다.
- 앱은 위기로 몰지 않고 안정 유지 중심으로 응답한다.

## 4. 해결 전략

### 접근 방식
- 입력 장벽을 낮추기 위해 자유 텍스트 한 줄 입력을 기본으로 한다.
- 응답은 공감, 상황 정리, 기술 1개, 실행 단계 중심으로 구성한다.
- 기술 라이브러리는 DBT의 핵심 스킬들로 제한해 일관성을 유지한다.
- 로컬 규칙 기반 응답과 OpenAI 기반 응답을 병행한다.

### 왜 이 방식인가
- 사용자는 감정 고조 시 긴 폼 입력이나 복잡한 설정을 견디기 어렵다.
- 완전한 상담 대체보다 "당장 다음 한 걸음"을 주는 것이 더 실용적이다.
- AI가 없더라도 앱이 최소 기능을 유지해야 실제 사용성이 높다.

## 5. 구현 내용

### 프런트엔드
- `index.html`: 모드 선택, 채팅 UI, 기술 카드, API 키 입력 UI 제공
- `styles.css`: 모바일 포함 반응형 스타일
- `app.js`: 모드 전환, 로컬 응답 로직, API 호출, 대화 렌더링 담당
- `manifest.json`, `sw.js`, `icon.svg`: PWA 설치 지원

### 백엔드
- `api/chat.js`: AI 응답 생성 API
- `api/config.js`: AI 사용 가능 여부와 모델 설정 확인 API
- `api/shared.js`: 공통 응답 처리, 프롬프트 구성, OpenAI Responses API 호출
- `server.js`: 로컬 개발용 정적 서버 + API 처리

### 동작 구조
1. 사용자가 한 줄 입력
2. 현재 모드와 대화 이력 확인
3. 로컬 응답 또는 AI 응답 엔진 선택
4. 감정 상태와 필요를 추정
5. 적절한 DBT 기술 1개와 실행 단계 반환

## 6. 문제 해결 결과

### 해결된 점
- 사용자는 짧은 입력만으로도 감정 정리 지원을 받을 수 있다.
- 진정, 마음챙김, 고통 감내, 감정 조절, 관계 정리 모드를 분리해 탐색 비용을 줄였다.
- AI 키가 없을 때도 로컬 모드로 기본 사용이 가능하다.
- 웹앱 형태라 설치 없이 접근 가능하고, PWA로 홈 화면 설치도 가능하다.

### 남아 있는 한계
- 이 앱은 의료행위나 치료를 대체하지 못한다.
- AI 응답 품질은 모델과 프롬프트 설계에 영향을 받는다.
- 위기 대응은 안내 수준이며 실시간 전문 개입을 제공하지 않는다.
- Vercel 배포 시 유효한 인증과 환경변수 설정이 필요하다.

## 7. 배포 관점의 현재 상태

### 확인된 사항
- 정적 파일 + `api/*.js` 구조라 Vercel 배포 구조와 맞다.
- `vercel.json`이 존재하며 serverless function 런타임 설정이 포함되어 있다.
- 프런트는 `./api/config`, `./api/chat` 경로를 사용하므로 배포 환경과 호환된다.

### 현재 배포 이슈
- 실제 배포 시 Vercel CLI 인증이 실패했다.
- 확인된 에러: `The specified token is not valid. Use vercel login to generate a new token.`

### 배포 완료 조건
- 유효한 Vercel 로그인 또는 `VERCEL_TOKEN`
- Vercel 프로젝트 연결
- `OPENAI_API_KEY` 환경변수 설정
- 필요 시 `OPENAI_MODEL` 설정
